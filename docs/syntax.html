Vapor Language

Vapor (vLang) is a language that sits on top of python

Home

Syntax

Get Interpreter

The syntax for Vapor is meant to be easy to use and remember, while also being intuitive. Also, since Vapor gets compiled into valid python code, you can write python code and vapor syntaxed code at the same time.

Curly Bracket Structure

As of v 0.0.1 curly brackets need to be on the same line for the interpreter to not throw a syntax error.

valid vapor syntax:

true == false {
  pass
}

invalid vapor syntax:

true == false
{
  pass
}

Printing Data

You can use the log method

log('Hello world')

   -> Hello World

But preferably, just start the line with single quotes

'Hello World'

   -> Hello World

An F string example using the single quote way

fruit = "apple"
'The fruit is an [[fruit]]'

   -> The fruit is an apple
expression = 100*5
'The expression evaluated to [[str(expression)]]'

   -> The expression evaluated to 500

Importing Vapor libraries

You can import your own .vap files into your code using the 'using' keyword.

The standard library wait.vap will be used in this example

using wait.vap

range(60) each number {
   wait(1)
}

If Statements

if statements are done by writing out a statement and putting true or false at the end:

number = 1

number == 1 true {
  'number is 1'
} else {
  'number is not 1, it's actually [[str(number)]]'
}

   -> 'number is 1'

Loops

Using the keyword 'each' you can determine a for loop

range(2) each number {
   'the current number is [[str(number)]]'
}

   -> the current number is 1
   -> the current number is 2

Python enumerate equivalence

The keyword 'with' will allow you to provide the index of the loop

letters = ['g','t','p']

letters each letter with curr_index {
   'the current letter: [[letter]] at index: [[str(curr_index)]]'
}
   -> the current letter: g at index: 0
   -> the current letter: t at index: 1
   -> the current letter: p at index: 2

Methods / Functions

Simply prefix the line with an m

m foo() {
   return "bar"
}

foo()

   -> bar

Try/Catch (aka Try/Fail)

You can use try statements. Still in elementary stages and subject to syntax change.

try {
   variable = "a" + 1
} fail {
   'we failed!'
}

Threads

You can easily call a method in a thread by prefixing the call with thread

m foo() {
   return "bar"
}
thread foo()

(foo would be ran in a thread separate from the main thread)

Classes

Create a class by prefixing the line with obj

a method statement with no name is interpreted as a constructor method ( m (_) )

an underscore after the first parenthesis is interpreted as 'self' in python and only works in the method statement

use self after using _, ie:

obj person {

   # constructor syntax below

   m (_, name, age) {
      self.name = name
      self.age = age
   }
}

# Create a list of person objects
people = [person("jhonny",19), person("debrah",32)]

people each human {
   'name: [[human.name]], age: [[human.age]]'
}

  -> name: jhonny, age: 19
  -> name: debrah, age: 32


Throw Errors

test = 1

test == 1 true {
   throw 'this is an error'
}

      -> this is an error

Built-in Benchmarking Parts of code

benchmark 'my benchmark'

range(10000) each number {
   number == 0 true {
   'beginning loop'
  }
}

benchmark end

   -> benchmark beginning my benchmark
   -> beginning loop
   -> benchmark time: 200 ms
